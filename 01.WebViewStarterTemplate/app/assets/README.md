# ⚡ ImpulseBin

**Заблокуй імпульсивні покупки. Збережи гроші.**

Мобільний додаток-«холодний душ» для шопоголіків.  
Коли хочеш щось купити — заблокуй це на N годин.  
Якщо бажання пройшло — гроші залишились у тебе.

---

## 📱 Збірка під Android

### Крок 1 — Встановлення залежностей

```bash
npm install
```

### Крок 2 — Ініціалізація Capacitor (лише один раз)

```bash
npx cap init "ImpulseBin" "com.yourcompany.impulsebin" --web-dir .
```

> ⚠️ Якщо `capacitor.config.json` вже є — цей крок пропускаємо.

### Крок 3 — Додати платформу Android

```bash
npx cap add android
```

### Крок 4 — Синхронізувати код з нативним проєктом

```bash
npx cap sync android
```

Цю команду повторюємо **кожного разу після змін** у HTML/CSS/JS.

### Крок 5 — Відкрити у Android Studio

```bash
npx cap open android
```

Натисни **Run ▶** у Android Studio для запуску на емуляторі або пристрої.

## 🌍 Локалізація (i18n)

Всі рядки — в об'єкті `i18n` всередині `app.js`.
Підтримується 7 мов: 🇺🇦 `uk` (за замовчуванням), 🇬🇧 `en`, 🇵🇱 `pl`, 🇩🇪 `de`, 🇫🇷 `fr`, 🇪🇸 `es`, 🇨🇳 `zh`.

Перемикач мови тепер живе на екрані **Налаштування** (`<select id="lang-select">`, клас `.lang-select-full`), а не в шапці. Вибір мови зберігається в `localStorage` (ключ `impulsebin_lang`) і відновлюється при наступному запуску.

Щоб додати ще одну мову:

```js
const T = {
  uk: { ... },
  en: { ... },
  pl: { ... },
  de: { ... },
  fr: { ... },
  es: { ... },
  zh: { ... },
  it: { ... }  // ← додай сюди + <option value="it">🇮🇹 Italiano</option> в #lang-select
};
```

Переключення програмно: `i18n.set('en')`, після чого виклич `applyStaticTranslations()` та перерендери список, щоб оновити вже відображені елементи.

## ⚙️ Налаштування (Settings)

Новий екран `#screen-settings` (іконка-шестерня в шапці) об'єднує:

- **Мову** — той самий `#lang-select`, перенесений з шапки.
- **Тему** — анімований перемикач `.theme-toggle` (Темна / Світла) із плавним
  ковзним повзунком (`.theme-slider`) на градієнтному золотому фоні.
- **Дані** — кнопка повного скидання прогресу.

### 🎨 Тема (Dark / Light)

```js
applyTheme('light'); // або 'dark'
```

Тема зберігається в `localStorage` (ключ `impulsebin_theme`) і застосовується
через атрибут `data-theme` на `<html>`. Світла тема — це не просто інверсія
кольорів, а окрема преміальна палітра (теплий ivory-фон із золотим
ґрадієнтом, `--gold: #C9971E`, м'які тіні) — дивись `:root[data-theme="light"]`
у `styles.css`. Усі ключові поверхні (header, картки, модалки, тости) мають
плавний `transition` між темами (~0.45s).

### 🗑️ Скидання прогресу

Кнопка в Налаштуваннях відкриває модалку підтвердження
(`#modal-reset-confirm`) і викликає `resetProgress()`, яка:

- очищує всі імпульси (`saveImpulses([])`) — обнуляє суму "Не витрачено";
- скидає бонусні рекламні слоти до 0;
- **залишає ачівки незайманими** (це історичні досягнення, а не "прогрес");
- розблоковує секретну ачівку *"Чиста сторінка"*.

## 🏆 Ачівки (40 штук)

Масив `ACH_DEFS` у `app.js` містить 40 досягнень у 7 категоріях:

| Категорія | К-сть | Приклад |
|---|---|---|
| `saved` — заощаджено $ | 6 | $100 → $5000 |
| `spent` — витрачено $ | 4 | $100 → $2500 |
| `bought_n` — куплено N разів | 3 | 1, 10, 25 |
| `cancel_n` — відмовлено N разів | 5 | 1 → 50 ("Дзен-майстер") |
| `created_n` — створено N імпульсів (будь-який статус) | 11 | 1 → 50 ("Легенда ImpulseBin") |
| `strict_n` — імпульсів у Суворому режимі | 1 | 5 |
| `pro` | 1 | перехід на Pro |
| секретні (`secret:true`) | 9 | див. нижче |

**Секретні ачівки** (назва й опис приховані до розблокування — екран
показує `❔` + локалізоване "Секретна ачівка" замість спойлера):

- 🎨 зміна теми вперше · 🌍 перепробував усі 7 мов · 💭 заблокував імпульс $1000+
- 🪶 створив імпульс на $0 · 📺 переглянув першу рекламу · 📡 вичерпав усі 7 бонус-слотів
- 🦉 створив імпульс після півночі · 🧹 скинув прогрес · 🏆 **"Колекціонер усього"** —
  мета-ачівка, що розблоковується, коли відкриті геть усі інші 39.

`created_n` рахує **всі** колись створені імпульси, незалежно від статусу —
тобто звичайний користувач вільної версії з лімітом у 3 слоти все одно
дійде до "create_50" з часом, просто завершуючи/скасовуючи старі імпульси.

## 📺 Реклама за додатковий слот (AdMob stub)

Free-тариф: **3** активних імпульси базово, +1 за кожен переглянутий
rewarded-ad, **максимум 10** (`FREE_TIER_LIMIT=3` + `MAX_EXTRA_SLOTS=7`).

Коли ліміт вичерпано — замість прямого "Перейти на Pro" з'являється модалка
вибору (`#modal-limit`): **"Переглянути рекламу (+1 слот)"** або
**"Перейти на Pro — без лімітів"**.

`showRewardedAd()` у `app.js` — заглушка (`setTimeout` ~1.8s), що імітує
повний цикл rewarded-реклами. Коментар у коді показує реальну інтеграцію:

```bash
npm install @capacitor-community/admob
npx cap sync
```

```js
import { AdMob, RewardAdPluginEvents } from '@capacitor-community/admob';
await AdMob.initialize({ requestTrackingAuthorization: true });
await AdMob.prepareRewardVideoAd({ adId: 'ca-app-pub-XXXXXXX/YYYYYYY' });
AdMob.addListener(RewardAdPluginEvents.Rewarded, () => grantExtraSlot());
await AdMob.showRewardVideoAd();
```

Бонусні слоти зберігаються в `localStorage` (ключ `impulsebin_ad_slots`,
clamped 0–7) і **скидаються до 0** при "Скинути прогрес".

## 📦 Data Structure

```ts
interface Impulse {
  id:         string;               // uid, 10 chars
  text:       string;               // опис покупки
  amount:     number;               // сума в гривнях
  createdAt:  number;               // Date.now() при створенні
  targetTime: number;               // Date.now() + затримка (ms)
  resolvedAt: number | null;        // Date.now() в момент complete/cancel (НЕ targetTime!)
  status:     'locked'              // таймер тікає
             | 'unlocked'          // час минув, чекає рішення
             | 'completed'         // куплено (гроші "збережено")
             | 'canceled';         // відмовився від покупки
  isStrict:   boolean;              // створено в Суворому режимі (без кнопки відмови)
}
```

> ⚠️ **Важливо:** `totalBlockingMs` у статистиці рахується як
> `(resolvedAt || targetTime) - createdAt`, а НЕ `targetTime - createdAt`.
> Це фікс бага, через який передчасна відмова/покупка раніше рахувалась
> так, ніби користувач чекав повний таймер.
